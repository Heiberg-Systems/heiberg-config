Unified app iconography shipped 2026-07-10. Each app has ONE canonical icon used on both public sites: the portfolio **motif** (accent-coloured) + the literal domain **glyph in a lower-right corner badge**. Composite geometry: viewBox `-200 -200 400 400`, badge `circle cx112 cy112 r82 stroke-width9 fill={surface}`, nested glyph `<svg x66 y66 92×92 stroke-width1.9>`. Per-app accents: Nourish `#3FB950`, Forge `#3B82F6`, Callout `#F0A500`, Stay `#FB923C`, Tally `#F472B6`, Heiway `#22D3EE`. Infra portfolio projects (Aide, IdP, Persistence) stay motif-only. Shared favicon = filled blue hexagon + knockout bolt on a `#0D1117` tile (replaced the Vite logo on personal-website and an off-brand purple mark on heiberg-systems-website).

Design: `docs/designs/2026-07-10-unified-app-iconography-design.md`; plan: `docs/plans/2026-07-10-unified-app-iconography.md` (root repo). PRs: heiberg-systems-website#45, personal-website#35, root pointer bump #224. Follow-ups: Our Apps card icons enlarged to 56px in the top-right corner (hsw#46); portfolio covers cache-busted (pw#36).

**Cover cache-busting (pw#36):** portfolio covers are external `<img src="/projects/<slug>.svg">` at STABLE urls, so a regenerated cover was served stale from browser/CDN cache (corporate icons are inline SVG in the JS bundle, so they bust automatically). Fix: `personal-website/frontend/src/utils/coverSrc.js` appends `?v=<__COVER_VERSION__>` (git short SHA injected via `vite.config.js` `define`, timestamp fallback) to local `/projects/*` urls only; applied at Portfolio/HomePage/ProjectDetail. So covers now self-bust on each deploy — no manual cache purge. Sites do NOT auto-deploy on develop merge; a redeploy is still what publishes any cover/icon change live.

**Per-app app-icon rollout (root #228, 2026-07-13):** each live app FRONTEND now carries its OWN icon (not the generic Heiberg hexagon) — motif-only `favicon.svg` + full-composite installed-app icons (192/512/maskable/apple), generated by a THIRD renderer `scripts/app-icons/gen-app-icons.mjs` (root repo, uses `sharp`; favicon=motif, install=composite, on the dark tile). Applied to nourish (leaf), stay admin/guest/discover (house), forge (bolt). **Forge's entire PWA contract (manifest + pass-through sw + PwaRegister + middleware exclusions) was scaffolded in the same PR** — it had none — closing the PWA-standard §7.2 retrofit. Amended `docs/designs/2026-06-29-platform-pwa-mobile-first-standard.md` from one-shared-hexagon → per-app composite; Callout/Tally/Heiway inherit when built. **Maskable PNG uses scale 0.65 (not the 0.8 default)** so the corner badge clears the 80% safe circle. Design/plan: `docs/designs|plans/2026-07-12-per-app-iconography*`.

**Why:** the same app used to look totally different per site (Nourish = leaf on one, fork+knife on the other), and favicons were off-brand.

**How to apply:** there is NO shared icon package — the same motif/glyph/accent registry is duplicated across THREE renderers: (1) `heiberg-systems-website/src/components/ProductIcon.jsx` (React, corporate slug e.g. `eating-plan`); (2) `personal-website/frontend/scripts/gen-project-covers.mjs` (build-time cover SVGs, product-name slug e.g. `nourish`); (3) `scripts/app-icons/gen-app-icons.mjs` (root repo, `sharp`, app-frontend favicon/PWA icons, product-name slug). When you change a motif/glyph/accent/geometry, update ALL THREE renderers and the design docs (`2026-07-10-unified-app-iconography` §4 + `2026-07-12-per-app-iconography` — the sole drift-control). Slug vocabularies differ between the repos — mind the mapping. Known cosmetic drift to fold back on next touch: corporate leaf outline has `strokeLinejoin="round"`, the personal generator's does not.
