**Centralized authentication & registration standard** ‚Äî 2026-07-07/08. Directive: no app authenticates/registers users itself; apps delegate sign-in to the central IdP (full-page redirect, SSO-silent) and only authorize on `{slug}:subscriber`. Register-first then subscribe. Mobile = credentials-first + opt-in biometric. Sign-out centralized (no per-app web sign-out; inactivity = realm SSO idle timeout; global sign-out on the website). Install (PWA/APK) gated behind entitlement.

- Design: `docs/designs/2026-07-07-platform-centralized-auth-standard.md` (amends PWA standard ¬ß4.1 ‚Äî drops "+ Sign out").

**Key decisions (baked into design/plans):**
- Web sign-in = full-page redirect to branded central IdP (not popup/broker). Apps keep per-app PKCE clients, own zero login/register UI.
- Biometric (mobile) = device-biometric-gated secure-enclave session; **declined biometric ‚áí re-login each launch** (unlockSession discards persisted session); passkeys deferred.
- Not-subscribed routing: Forge + Stay admin (whole-app 403) ‚Üí redirect to website subscribe page; **Nourish EXCLUDED** (partial gate, 402 on AI only ‚Äî keeps free access, soft CTA deferred).
- The web login "delete the page" is really a zero-interaction redirect **shim** (Auth.js needs a route for PKCE/state).
- Register-link repoint moved plan 1‚Üí**plan 2** (registration already central on Keycloak, not in apps).
- **stay-guest role granted by BOOKING-ASSOCIATION** (decided 2026-07-07): non-role-gated `POST /me/claim-guest` in stay-api verifies a verified-email booking, links it, calls a new portal-api allowlisted internal `/internal/users/{sub}/grant-role` (stay-guest only, never a subscription role); guest SSO-re-auths to refresh the token. Fixes a pre-existing gap (nothing grants stay-guest today).
